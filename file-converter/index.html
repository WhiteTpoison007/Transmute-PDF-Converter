Under the Hood

How does this actually work?

No magic. No servers. Just your browser doing things most people don't know it can do.

The key idea

Your browser is already a powerful computer

When you visit a website, you're not just downloading pretty pictures and you're running a full program. JavaScript (the language that makes websites interactive) can read files, process data, draw graphics, and create new files entirely inside your browser tab. We just tap into those built-in powers.

Think of it like a calculator app so it doesn't need to call a server every time you press a button. It just runs the math locally. TRANSMUTE does the same, but with files.

The 3 free tools powering this
📖
PDF.js
by Mozilla (Firefox folks)

Normally browsers can display PDFs but can't let code manipulate them. PDF.js unlocks that: it reads the PDF's internal structure and lets us pull out pages, text, and graphics.

Used for: PDF → Images, PDF → Text, PDF → HTML
🔨
pdf-lib
open source

Does the opposite of PDF.js: it builds PDFs from scratch. When you pick images and hit convert, pdf-lib is assembling a brand new PDF document in your computer's memory, page by page.

Used for: Images → PDF
📦
JSZip
open source

When a PDF has 10 pages and you convert them all to images, you'd normally get 10 separate downloads. JSZip bundles them into a single .zip file, all inside the browser.

Used for: Multi-file downloads
What happens when you convert a PDF to images
01
You pick a file
The browser reads it into memory (RAM) using a built-in API called FileReader. The file is now just data, a bunch of bytes, sitting in your browser tab's memory. It hasn't gone anywhere.
02
PDF.js parses the file
PDF is a complex format with its own language inside. PDF.js reads it like a translator: it figures out where each page is, what fonts are used, where images live, and where text is positioned.
03
Pages are drawn on an invisible canvas
The browser has a built-in drawing surface called a <canvas>, think of it as an invisible whiteboard. PDF.js "paints" each page onto it, exactly like how your screen renders graphics in a video game.
04
Canvas → Image (Blob)
The canvas gets converted to a Blob: raw file bytes in memory. A Blob is like holding a photo in your hands. It exists, but hasn't been saved to disk yet. Think of RAM vs hard drive.
05
A fake download link is created
JavaScript creates a temporary URL pointing to that Blob (like blob://page-1.png), attaches it to an invisible link, and clicks it automatically. The browser saves the file to your Downloads folder.
06
Memory is freed
Once downloaded, the Blob URL is revoked and memory is cleared. Clean, fast, private. Nothing lingers.
💡
Why does it need internet on the first load?
The three libraries (PDF.js, pdf-lib, JSZip) are loaded from a CDN, a fast file-hosting service. After the first visit, your browser caches them (saves a local copy), so the site works fully offline from then on. Your actual files are never sent anywhere.
